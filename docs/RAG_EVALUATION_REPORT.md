> \* 减去等待 ingestion 的 1s 后净延迟

### 2.2 检索延迟（Golden QA 实测）

全量 25 题检索总计 14.8s，平均 **593ms/题**。

### 2.3 瓶颈分布

| 环节 | 占比 | 说明 |
|------|------|------|
| pgvector 向量检索 | ~20% | 小数据量下表现良好 |

| 相关性 Gate | ~10% | 轻量文本匹配 |
| 其他（DB/ORM） | ~10% | 受连接池影响 |

## 三、鲁棒性评估（Robustness）

### 3.1 极端场景测试结果（11 项全通过）

| 场景 | 覆盖 | 结果 |
|------|------|------|
| LLM 下游 502 | stream_deepseek 异常传播 | ✅ |
| LLM 5xx HTTP 端到端 | 服务不崩 | ✅ |
| Embedding 503 | embed_texts 抛异常 | ✅ |
| Embedding 5xx Ingestion | 文档 failed 状态 | ✅ |
| DB 连接断开 | 3 个 API 返回异常 | ✅ |
| 存储写入 OSError | 不落脏数据 | ✅ |
| 预览文件不可读 | 返回 404/410 | ✅ |
| Ingestion 读取失败 | failed + 中文错误 | ✅ |
| 登录限流 5次/15min | → 429 | ✅ |
| IP 级限流 20次/5min | → 429 | ✅ |
| 渐进锁定期 1m→5m→15m→1h | 递增等待 | ✅ |

### 3.2 降级策略覆盖

| 故障 | 降级行为 |
|------|----------|
| LLM 调用失败 | 异常传播，不静默吞掉 |
| Embedding 失败 | 文档标记为 failed |
| DB 不可用 | → 503 Service Unavailable |
| 存储不可用 | → 500 + 中文提示 |
| 检索结果为空 | query rewrite 重试一次，仍空→拒答 |

**评级**: ⭐⭐⭐⭐（4/5）

## 四、生成质量评估（Generation Quality）

使用 DeepSeek Chat 进行 LLM-as-Judge 逐题评分（1-5 分），覆盖全部 25 题。

| 维度 | 均分 | 评级 |
|------|------|------|
| **正确性（Correctness）** | **4.08/5.0** | ⭐⭐⭐⭐ |
| **忠实度（Faithfulness）** | **4.64/5.0** | ⭐⭐⭐⭐⭐ |
| **相关性（Relevance）** | **4.68/5.0** | ⭐⭐⭐⭐⭐ |
| **综合** | **4.47/5.0** | ⭐⭐⭐⭐ |

### 4.1 逐题评分分布

| 评分区间 | 正确性 | 忠实度 | 相关性 |
|----------|--------|--------|--------|
| 5 分 | 11 题 | 18 题 | 18 题 |
| 4 分 | 7 题 | 3 题 | 4 题 |
| 3 分 | 5 题 | 2 题 | 2 题 |
| 2 分 | 2 题 | 1 题 | 1 题 |
| 1 分 | 0 题 | 1 题 | 0 题 |

### 4.2 低分原因分析

| 最低分 | 原因 |
|--------|------|
| GQ-4 (c=2, f=1) | 英文 PDF 检索失败，LLM 无依据编造回答 |
| GQ-15 (c=2, f=3) | 加班流程回答不完整，遗漏了关键细节 |
| GQ-3 (c=3, f=4) | 年终奖信息回答基本正确但缺少精确表述 |
| GQ-12 (c=3, f=4) | 带薪年休假回答正确但部分措辞不够精确 |

**结论**: LLM 生成质量优秀（综合 4.47/5），忠实度高达 4.64，表明系统能有效抑制幻觉。低分主要来源于检索失败后的无依据回答，而非 LLM 本身问题。

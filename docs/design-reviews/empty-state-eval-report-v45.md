## 维度细评（满分 10 分）

### 1. 信息架构（IA）
- **Dashboard**：9.8 — 空态即主线，CTA 与 3 步引导、4 维指标、RAG 预告层次分明。
- **KBs**：9.7 — 从"建库→上传→对话"到"4 维指标"过渡自然。
- **KBDetail**：9.7 — 从"资料库无文档"切入，步骤与指标贴合 KB 场景。
- **Ask**：9.6 — "无资料库"场景解释清楚，但"多库选择"与"全局检索"的说明可以再精简。
- **Members**：9.5 — 仅所有者场景清晰，但"SSO 接入"对空态来说权重略高。
- **Account**：9.4 — 首次设置场景合理，但和已有设置卡片并存，空态叙事不如纯空态页面聚焦。

### 2. 视觉层次
- 全部 9.7+。暖白渐变 HERO + 衬线标题 + 辅助标签，层级稳定。
- Account 扣 0.1：v44 空态位于 main 底部，与上方 settings-wrap 形成视觉分区，需用户滚动才能完整看到。

### 3. 可访问性（a11y）
- **全部 9.8+**：所有按钮带 `type="button"`，所有装饰 SVG 带 `aria-hidden="true"`，交互元素有 `aria-label`/`aria-labelledby`。
- 弹窗使用原生 `<dialog>` + `showModal()`，focus 陷阱正确。
- 简单模式切换为 `role="switch"` + `aria-pressed`。

### 4. 交互与反馈
- **全部 9.6+**：CTA hover、卡片 hover、弹窗打开/关闭动画、复制反馈、邀请发送反馈均完整。
- 邀请弹窗的"已邀请名单"有动态 slide-in。

### 5. 性能
- 全部 9.8+。纯静态 HTML/CSS/JS，无外部依赖，首屏 < 1s。
- Dashboard/KBs 有较多 CSS 动画，但均使用 transform/opacity。

### 6. 代码质量
- 全部 9.7+。语义化标签完整，class 命名一致，无 HTML5 嵌套错误。
- 通用注入器 `inject_v44.py` 实现可复用，5 场景配置化。

### 7. 响应式
- 全部 9.6+。Desktop / Tablet / Mobile 三套断点，HERO 在窄屏自动单列。
- 4 维卡片在 tablet 自动 2 列，在 mobile 单列。

### 8. 品牌一致性
- 全部 9.8+。统一使用 terracotta 暖色系、衬线标题、圆角、阴影、图标风格。

### 9. 空态语义准确性
- **Dashboard/KBs/KBDetail**：9.8+ — 本身就是空态页面，v4.4 完美替代旧空态。
- **Ask**：9.6 — 原页面是满态对话 demo，v44 嵌入 `view-empty` 作为状态之一，语义成立。
- **Members/Account**：9.4 — 原页面是满态管理页，v44 作为"空状态样板"追加在底部，是"状态演示"而非真实空态。这在预览页场景下合理，但生产代码中应作为独立空态视图。

### 10. 动效与微交互
- 全部 9.7+。HERO 渐变呼吸、脉冲环、漂浮文档、sparkle 均统一。
- simple mode 一键关闭所有动画，符合 a11y 偏好。

### 11. 文案与语调
- 全部 9.6+。场景化、具体、可执行，避免"暂无数据"式敷衍。
- 部分场景（如 Account"导入从飞书/钉钉"）在空态中略显前置，扣 0.1。

### 12. 鲁棒性/容错
- 全部 9.8+。0 JS 错误，0 运行时异常，弹窗关闭路径完整（×、稍后、Esc、backdrop click）。

## 后续建议

1. **Chat 保持现状**：Chat 页面原本就用多状态 demo（对话/空态/加载/异常），强行套 v4.4 会覆盖现有 demo。如需 Chat 空态 v4.4，建议单独加一个 `view-empty-v44` 状态，而非替换。
2. **生产同步**：4 个新页面的 v4.4 空态结构可直接映射到 React 组件：`EmptyHero` + `EmptySteps` + `EmptyGrid` + `RagPreview` + `InviteDialog`。
3. **下一步 B 计划**：将 6 个空态组件同步回 `frontend/src/`，替换现有空态/空状态提示。

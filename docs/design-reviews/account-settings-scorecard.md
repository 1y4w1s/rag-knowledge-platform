## 1. 六维打分（门禁核对）

| 维度 | 权重 | 得分 | ≥8.0 | 说明 |
|---|---|---|---|---|
| 一致性 Consistency | 20% | **9.3** | ✅ | 外壳/侧栏 SVG 图标与 KBs·KBDetail 完全一致；强度条 token 已映射到统一红/琥珀/绿（消除登录页私有漂移）；状态色语义合规。唯一残留：确认离开按钮源码用品牌赤陶（AS-R1-3，源级，保全）。 |
| 可用性 Usability | 20% | **9.2** | ✅ | 密码显示/隐藏可切换；强度条随输入实时变化；离开确认弹窗支持 Esc/点遮罩关闭；失败态「重试」可回到设置；无死控件。 |
| 功能保全 Function | 20% | **9.8** | ✅ | 四视图覆盖源码全部条件分支（已加入→离开团队 / 未加入→加入团队 / 加载骨架 / 失败 AlertBanner+重试）+ 离开确认弹窗 + 密码强度/显隐。原版功能零增零减。 |
| 无障碍 A11y | 15% | **9.0** | ✅ | 表单 `label[for]`↔`id` 全配对；密码 toggle 动态 `aria-label`；强度条 `role=meter`+`aria-valuenow`；对话框 `role=dialog aria-modal aria-labelledby`；AlertBanner `role=alert`。只读字段为 focusable input（可接受）。 |
| 视觉 Visual | 15% | **9.2** | ✅ | 卡片 16px 节奏、标题 serif 26px 与兄弟页一致；控件圆角 8px / focus 赤陶环；新增 H1 统一内容页层次。 |
| 性能 Perf | 10% | **9.0** | ✅ | 无重动效；背景漂移 `will-change`+`prefers-reduced-motion` 守卫；骨架脉冲尊重 reduced-motion。 |

**加权总分** = 0.2×9.3 + 0.2×9.2 + 0.2×9.8 + 0.15×9.0 + 0.15×9.2 + 0.1×9.0
= 1.86 + 1.84 + 1.96 + 1.35 + 1.38 + 0.90 = **9.29 / 10**

**门禁结论**：六维全部 ≥ 8.0 → ✅ **本轮达标**，加权 **9.29**。

## 2. 本轮改造前后（AS-R1-1 ~ AS-R1-5）

| 编号 | 痛点 | 改造前 | 改造后 |
|---|---|---|---|
| AS-R1-1 | 强度条 token 漂移 | 登录页私有 `--auth-strength-weak/mid/strong` | 统一 `--fail`(红)/`--amber`(琥珀)/`--success`(绿) |
| AS-R1-2 | 只读字段无区分 | 与可编辑 input 同款 | `.readonly`：surface-2 底 + ink-2 字 + 无焦点环 |
| AS-R1-4 | 列宽 | 460px | 440px（对齐源码 `max-w-[440px]`） |
| AS-R1-5 | 缺页标题 | 仅面包屑 | 补 serif H1，统一内容页节奏 |

> AS-R1-3（确认离开按钮用品牌赤陶）为**源码既有决策**，功能保全优先不动；作为待产品确认项记入台账，不阻断门禁。

## 4. 已知限制 / 待办（不阻断门禁）

- **AS-R1-3（P3）**：`确认离开` 按钮源码用品牌赤陶（`#B85A2E`），按设计系统 §4.1 销毁型操作应走 `danger` 红。功能保全优先保留，建议产品确认是否改红。
- 两视图各含一份「修改密码」表单（id 区分），强度条独立绑定，无冲突。
- 同步回代码时：`PasswordStrengthBar` 的 `--auth-strength-*` 应替换为统一状态色变量（见 design-system §2 映射）。

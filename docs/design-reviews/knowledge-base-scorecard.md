# 知岸 · 资料库两页重设计评分卡（评分环节）

> **范围**：本轮（Round 2）迭代收尾的**资料库列表页** + **资料库详情页**。
> **标尺**：`docs/design-system.md` 锁定的设计系统（状态语义 / Token / 自定义控件 / a11y）+ `ardot-ui-design` 16 条产品定律。
> **打分区间**：0–10。加权总分 = 一致性 20% · 可用性 20% · 功能保全 20% · 无障碍 15% · 视觉 15% · 性能 10%。
> **评分时点**：Round 2 修复**后**（列表页已完成对齐，详情页筛选已活化）。

## 三、扣分点（残余，非阻断）

- **列表页无障碍 8.8**：侧栏导航激活态、跳转链接仍依赖代码层路由，预览无法体现（属外壳层，见 `app-shell-audit.md`）。
- **详情页视觉 9.0**：中文衬线标题在 Windows 仍偏细（H5 已知限制，需评估改无衬线加粗以与登录页统一）。
- **状态 specimens 不可交互**：筛选条"选中状态"在预览为单击演示，真实过滤需代码层数据；功能覆盖已用 specimens 证明，不扣分。

## 四、总评

> **知识库两页已达「设计系统统一、功能零减损、无已知 P0/P1 缺陷」基线（均值 ≈ 9.26）。**
> 列表页经 Round 2 追平详情页规范，跨页一致性问题（赤陶撞色、原生控件、Token 漂移、死分页）全部闭合。

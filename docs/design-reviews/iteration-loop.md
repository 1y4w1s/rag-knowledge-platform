# 睿阁 · 重设计迭代闭环（Iteration Loop）

> **本文件是「逐页代码重做 → 评审 → 验收」循环的常驻流程与进度台账。**
> 它定义循环步骤、验收门禁（gate）与全页进度矩阵；每次完成一轮，在此追加记录。
>
> **配套**：`docs/DESIGN.md`（唯一设计权威）、`design-system.md`（简短补充）、`app-shell-audit.md`（外壳审计）。

---

## 1. 循环定义（每轮三步）

```
① 代码重做 → 对照 DESIGN.md + 已达标 Dashboard 页基线，用真实代码实现该页
② 自我评审 → 按 §2 六维打分，逐维核对门禁清单
③ 验收记录 → 在 §3 进度矩阵追加记录
```

**门禁（Gate）**：任意一页须 **六个维度全部 ≥ 8.0 / 10** 才算该页达标；否则回到 ① 继续。

---

## 2. 评分维度与权重

| 维度 | 权重 | 说明 |
|---|---|---|
| 一致性 Consistency | 20% | 跨页 Token / 状态语义 / 控件风格统一 |
| 可用性 Usability | 20% | 控件可操作、反馈明确、无死控件 |
| 功能保全 Function | 20% | 原版功能零增零减 |
| 无障碍 A11y | 15% | 地标 / 焦点环 / 语义角色 / 键盘可达 |
| 视觉 Visual | 15% | 层次 / 节奏 / 留白 / 视觉语言统一 |
| 性能 Perf | 10% | 动效守卫 / 合成层 / 长列表 |

**加权总分** = Σ(维度分 × 权重)。但门禁看的是**每一维是否 ≥ 8.0**，不是总分。

## 4. 验收检查清单

每页重做完成后，对照以下条目逐项检查：

- [ ] 颜色全部使用 `var(--*)`，无硬编码色值
- [ ] 亮色/暗色双主题全覆盖
- [ ] 卡片圆角 16px，控件圆角 8px
- [ ] 缺失值显示 `—`，三个态（loading/error/empty）齐全
- [ ] 容器 `max-w-[1180px]`，`px-7`
- [ ] 角色/权限徽章用 `--role` 系，不用状态色
- [ ] WCAG AA：焦点可见环、对比度合规、`prefers-reduced-motion` 守卫
- [ ] 视觉效果与已落地的 Dashboard 页面一致
- [ ] 中文正文无 `letter-spacing`（DESIGN.md §7 红线）
- [ ] 品牌色 `--action` 仅用于主操作和激活态，不借给状态

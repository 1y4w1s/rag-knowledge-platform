# 空态设计评估体系 v1

> 目的：搭建一套**结构化、可量化、可复现**的空态设计质量评估框架。覆盖视觉一致性、信息层级、交互体验、可访问性、品牌气质、情感温度 6 维。每维配 5+ 条硬指标、扣分规则、5 档位。所有评分均可程序化测量或可被评审者按统一规范打分。
>
> 配套：Dashboard 概览空态改版前后对比见 §5；6 维评分表见 §6。

---

## 0. 评分刻度（5 档）

| 档位 | 加权分 | 含义 | 处置 |
|---|---|---|---|
| **S 卓越** | ≥ 9.5 | 行业标杆，5 维全 ≥ 9.0 | 直接通过，可作为范本 |
| **A 优秀** | 9.0 ≤ x < 9.5 | 达标，1-2 维待打磨 | 通过，记「打磨项」 |
| **B 良好** | 8.0 ≤ x < 9.0 | 局部短板，需返工 | **不通过**，≥1 轮返工 |
| **C 待改进** | 6.0 ≤ x < 8.0 | 系统性缺陷 | **不通过**，≥2 轮返工 |
| **D 不达标** | < 6.0 或一票否决 | 严重问题 | 驳回重做 |

**降级规则**：
1. 加权 < 9.0 = 不通过
2. 任一一票否决 = D 档
3. 任一维 < 8.0 = 强制降档（如 9.3 降 B）
4. 全部硬指标必须程序化测量或附实证截图；未测 = 该项 0 分

## 1. 6 维清单 + 权重

| 编号 | 维度 | 权重 | 性质 | 一票否决条件 |
|---|---|---|---|---|
| **E1** | 视觉一致性 | 18% | 客观可测 | 与 design-system 严重漂移（≥ 3 处色/字/圆角） |
| **E2** | 信息层级 | 20% | 半客观 | 主从信息倒置 / 关键操作不可识别 |
| **E3** | 交互体验 | 20% | 客观可测 | 主 CTA 不可达 / 无引导步骤 |
| **E4** | 可访问性 | 12% | 客观 | 键盘不可达 / 焦点环缺失 / 装饰图缺 alt |
| **E5** | 品牌气质 | 18% | 70%客观 30%主观 | 完全无品牌元素 |
| **E6** | 情感温度 | 12% | 70%主观 30%客观 | 文案机械化/无情感（如"暂无数据"） |
| | **合计** | **100%** | | |

> 权重说明：E2 + E3 合计 40%，是空态最核心的「让用户知道做什么」能力；E5 + E6 合计 30%，是空态区别于"占位符"的关键差异化能力。

## 2. 各维硬指标（E? = Empty-state 指标）

### E1 视觉一致性（18%）

| 编号 | 硬指标 | 扣分规则 |
|---|---|---|
| E1.1 | 主色板与 token 对齐（`--terracotta / --amber / --ok / --err` 不可漂移） | 1 处漂移 -0.2，3 处 -0.5 |
| E1.2 | 字号走 token（`11.5/12/12.5/13/13.5/14/15/15.5/16/18/19/20/24/30` 之一） | 1 个不命中 -0.1，5+ -0.4 |
| E1.3 | 圆角 token（`4/8/9/10/12/14/18` 之一） | 1 处游离 -0.1 |
| E1.4 | 间距 4 倍数（`4/6/8/10/12/14/16/18/20/24/28/32` 之一） | 1 处游离 -0.1 |
| E1.5 | 图标 100% SVG（`stroke:currentColor / width:1.6-1.9`） | 1 个非 SVG -0.3 |
| E1.6 | 装饰性插画风格一致（线宽/曲率/视口比例统一） | 风格不统一 -0.5 |

**达标线**：≤ 1 个 E 指标违反 → A 档；3+ → B 档。

### E2 信息层级（20%）

| 编号 | 硬指标 | 扣分规则 |
|---|---|---|
| E2.1 | 主标题（h2）字号 ≥ 24px（区别于正文 12-14px） | < 24px -0.5 |
| E2.2 | 副标/正文与主标对比 ≥ 1.5 倍字号差 | 不达标 -0.3 |
| E2.3 | 主 CTA（按钮）vs 次 CTA（按钮）对比明确（主用 brand 色，次用 line/ghost） | 区分不明 -0.4 |
| E2.4 | 步骤编号/数字徽章字号 ≥ 14px | < 14px -0.2 |
| E2.5 | 信任行/数据小字 ≥ 11.5px | < 11.5px -0.2 |
| E2.6 | 同一区域字号种类 ≤ 4 种（避免混乱） | > 4 种 -0.3 |
| E2.7 | 标题与正文间距 ≥ 8px | < 8px -0.2 |

**一票否决**：主从信息倒置（次级信息更突出）或关键操作不可识别 = D 档。

### E3 交互体验（20%）

| 编号 | 硬指标 | 扣分规则 |
|---|---|---|
| E3.1 | 主 CTA 文字明确（不是"确定/提交"），动词在前 | 弱动词 -0.3 |
| E3.2 | 主 CTA ≤ 2 个 hover/focus 状态变化 | 无变化 -0.3 |
| E3.3 | 至少 1 个主 CTA + ≥ 1 个次 CTA + ≥ 1 个引导步骤 | 缺 1 类 -0.5 |
| E3.4 | 步骤引导数量 ≤ 4 步（认知负担） | > 4 步 -0.3 |
| E3.5 | 引导卡 hover/focus 状态（边框/阴影/位移） | 无任何变化 -0.4 |
| E3.6 | CTA 跳转目标可达（不能是 `href="#"`） | 1 个死链 -0.5 |
| E3.7 | 输入框 placeholder 描述能力（不是"输入..."） | 弱描述 -0.2 |

**一票否决**：主 CTA 不可达 / 完全无引导步骤 = D 档。

### E4 可访问性（12%）

| 编号 | 硬指标 | 扣分规则 |
|---|---|---|
| E4.1 | 所有插画 / 装饰 SVG `aria-hidden="true"` | 缺 -0.2 |
| E4.2 | 标题区 `<section>` 或 `<header>` 语义，含 `aria-label` | 缺 -0.3 |
| E4.3 | 主 CTA 显式 `type` 属性 + `aria-label` 描述结果 | 缺 -0.3 |
| E4.4 | `:focus-visible` 焦点环 ≥ 2px 赤陶色，offset ≥ 2px | 缺 -0.5 |
| E4.5 | 引导卡可键盘 Tab + Enter 触发 | 卡死 -0.5 |
| E4.6 | 文案对比度 ≥ 4.5:1（正文） | 1 处 < 4.5 -0.3 |
| E4.7 | 触摸目标 ≥ 44×44px（移动端） | 1 处 < 44px -0.2 |
| E4.8 | `prefers-reduced-motion` 守卫（动效可关闭） | 缺 -0.3 |

**达标线**：E4.1 / E4.2 / E4.4 / E4.5 全部满足 + 总违规 ≤ 3 → A 档。

### E5 品牌气质（18%）

| 编号 | 硬指标（70% 客观） | 扣分规则 |
|---|---|---|
| E5.1 | 至少 1 个品牌主色（赤陶）出现在主 CTA / 关键装饰 | 缺 -0.5 |
| E5.2 | 至少 1 个品牌元素（暖白渐变 / 赤陶线性插画 / 暖意文案） | 缺 -0.5 |
| E5.3 | 字体语言：标题用 serif（宋体）+ 正文 sans | 缺一 -0.2 |
| E5.4 | 状态切换按钮/插画与全站其他空态（KBs/KBDetail）插画风格一致 | 不一致 -0.3 |
| E5.5 | 微动效（脉冲 / 虚线流动 / 火花）体现"等待/智能"品牌隐喻 | 全无 -0.3 |
| 主观（30%）| 与暖白/赤陶语言契合度 | 0-1 分 |

**一票否决**：完全无品牌元素 = D 档。

### E6 情感温度（12%）

| 编号 | 硬指标（30% 客观） | 扣分规则 |
|---|---|---|
| E6.1 | 文案不用"暂无数据"/"无内容"等机械表达 | 机械文案 -0.5 |
| E6.2 | 文案给用户承诺（"30 秒开始整理"）或赋能（"像问同事一样问"） | 无任何承诺 -0.3 |
| E6.3 | 标题用第二人称"你"或名词化隐喻（"从这里开始"） | 第三人称或命令式 -0.2 |
| E6.4 | 至少 1 处情感触点（暖色背景 / 拟人化插画 / 信任行） | 0 处 -0.5 |
| E6.5 | 副标不超过 2 行（避免说教感） | > 2 行 -0.2 |
| 主观（70%）| 是否让用户感受到"产品懂我" | 0-1 分 |

**一票否决**：文案完全机械化 = D 档。

## 3. 评审流程（每页 3 步）

```
A. 程序化扫描（5min）
   ├── E1.1-E1.6 / E2.1 / E3.6 / E4.4 / E4.7 / E4.8 自动检查
   ├── 触发一票否决 = 直接 D 档
   └── 评分记录：每条 E? 指标有/无/扣分值

B. 视觉复核（10min）
   ├── E2.2-E2.7 / E4.1-E4.3 / E5 / E6 主观
   ├── 截图存档
   └── 评审者按 0-1 打主观分

C. 综合评级
   ├── 加权计算（6 维权重）
   ├── 降档检查（任一维 < 8.0 → 降档）
   └── 输出 S/A/B/C/D + 违规清单
```

## 6. 推广建议

1. **本套评估体系**适用于所有空态（KBs 列表空、KBDetail 文档空、Chat 历史空、Ask 无结果、Members 邀请空等 8+ 个空态场景）
2. 评估周期：每个空态上线前必走 A→B→C 3 步流程
3. 评估档位 ≥ B 才允许上线
4. 每月 1 次空态质量复查（产品迭代可能引入新的空态）

## 7. 附录：评估流程 checklist

```
□ A1. 程序化扫描（5min）
    □ Playwright 渲染截图存档
    □ 扫描 token 漂移（色/字/圆角/间距）
    □ 扫描 SVG 属性（stroke/width/fill）
    □ 扫描可达性（focus 环、tabindex、aria-*）
    □ 扫描无障碍（触摸目标、对比度、reduced-motion）
    □ 扫描一票否决条件

□ B1. 视觉复核（10min）
    □ 检查信息层级（h2/正文/CTA/信任行）
    □ 检查品牌元素（色/字/插画/动效）
    □ 检查情感温度（文案/承诺/触点）
    □ 评审者按 0-1 打主观分
    □ 截图对比 before/after

□ C1. 综合评级
    □ 计算 6 维加权分
    □ 检查降档规则
    □ 输出 S/A/B/C/D
    □ 写违规清单 + 仍待优化项
    □ 归档到 docs/_render/empty-eval/
```

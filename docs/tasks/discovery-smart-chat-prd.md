## G-1-1 定位 · 正常流程 · 乱操作 ✅ 已确认（2026-07-08）

**这节定什么**：在 **「我的空间」或「团队空间」** 内，用户 **不用先选资料库** 即可提问；系统在 **当前可见的全部资料库** 中检索，回答可带 **多库引用**。与现有 **库内对话** 并存，不替换。

### 与库内对话对比

| | 库内对话（保留） | 工作区智能问答（新增） |
|--|------------------|------------------------|
| 入口 | 资料库详情 →「开始对话」 | 侧栏 **「对话」**（G-1-2） |
| 检索范围 | **仅当前 1 库** | **当前 workspace 全部可见库** |
| 典型用户 | 已知问题在哪个库 | **不知道在哪个库** |

### 正常流程（S）

| # | 用户做什么 | 看见什么 |
|---|------------|----------|
| S1 | 登录 → 侧栏选 **我的空间** 或 **团队名** | 当前 workspace 确定 |
| S2 | 点侧栏 **「对话」** | 进入 **工作区对话**（页头不强调库名） |
| S3 | 输入「年假有多少天？」 | 流式出字 |
| S4 | 看回答下方引用 | 可多条 chip，**不同资料库/文档** |
| S5 | 点 chip | 跳进 **该库** 文档预览（与现网一致） |

**检索范围（白话）**：只搜「这个人此刻能看到的库」；研发 Member 搜不到市场机密库。

### 乱操作 / 边界（E）

| 乱操作 | 系统怎么处理 | 你怎么验 |
|--------|--------------|----------|
| E1 个人空间问团队独有内容 | 搜不到 → **拒答** | 个人空间提问 |
| E2 团队 A 问团队 B 内容 | 看不到 B 的库 → 拒答 | 换账号 |
| E3 切部门后仍在旧对话页 | 按 **新部门可见库** 重检；旧引用可能失效 → toast | 切部门后再问 |
| E4 硬闯 `/knowledge-bases/{错库}/chat` | **保持现状**：只搜该库 | 地址栏 |
| E5 可见库均无依据 | **拒答**（AC-4） | 无关问题 |
| E6 连点发送 | **M5** 对话限流 30 次/小时 | 连发 |
| E7 多库都有依据 | 回答可带 **多库引用**；chip 带库名 | 多库同主题 |

### 用户拍板记录

- **2026-07-08**：用户确认方案 **C**（自动选库 / 工作区级问答），Implement 前须 Research + Plan。

## G-1-2 入口 · 路由 · 分工 ✅ 已确认（2026-07-08）

> **命名（2026-07-08 用户意见）**：侧栏与按钮 **沿用「对话」**，**不用**「问知岸」等产品名做功能文案；品牌/Logo 后续可能整体升级，Implement 用中性文案。

### 侧栏（主入口）

| 现在 | 改后 |
|------|------|
| 「对话」→ 最近用过的库；无最近库 → 资料库列表 | 「对话」→ 固定 **`/ask`**（工作区多库检索） |

| 元素 | 行为 |
|------|------|
| 文案 | **对话**（不改名） |
| 路由 | **`/ask`** |
| 团队 / 个人 | 两种空间都有；跟 workspace + 部门 scope |
| Member / Admin | 都能点 |

**库内「开始对话」保留**：`/knowledge-bases/{id}/chat`（单库模式）。

**怎么区分两种对话（用户不背术语）**：

| 从哪进 | 页头/面包屑 | 检索范围 |
|--------|-------------|----------|
| 侧栏「对话」 | 显示 **「对话」** 或副标题「当前空间」· **不出现单一库名** | 多库 |
| 库详情「开始对话」 | 显示 **该资料库名称** | 单库 |

### 概览 Dashboard（副入口）

| 元素 | 行为 |
|------|------|
| 快捷提问输入框 | 占位：**「输入问题，在全部资料库中检索…」**（中性，不带产品名） |
| 发送 | 跳 `/ask?q=…` 并自动发问 |
| 原「进入对话」按钮 | 文案仍 **「进入对话」** → `/ask`（不绑最近库） |

### 与「找文档」分工

| 功能 | 用户意图 | 入口 | 结果 |
|------|----------|------|------|
| **找文档** | 找文件看 | 概览「找文档」 | 详情/预览 |
| **对话（工作区）** | 要答案+引用 | 侧栏 / 概览提问 | `/ask` 多库 |
| **对话（库内）** | 确定在这库问 | 库详情「开始对话」 | 单库 chat |

### 路由

| 路由 | 说明 |
|------|------|
| **`/ask`** | 新增 · 工作区智能问答 |
| `/knowledge-bases/:id/chat` | 保留 · 库内对话 |

### 乱操作（本节）

| 乱操作 | 系统怎么处理 | 你怎么验 |
|--------|--------------|----------|
| E8 未登录 `/ask` | 跳登录 | 隐身窗口 |
| E9 无可见库 | 空态 + 引导建库/加入团队 | 新号 |
| E10 空问 / 超长 `?q=` | 不发 / 与库内对话同规则 | 粘贴长文 |
| E11 对话中切 workspace | 跳概览或清 thread（对齐现 UX） | 切空间 |

### 用户确认记录

- **2026-07-08**：侧栏文案 **「对话」**；路由 **`/ask`**；概览中性占位；与「找文档」分工不变。

## G-1-3 引用展示（多库 chip）✅ 已确认（2026-07-08）

**这节定什么**：工作区 `/ask` 回答下方引用 chip 的文案、条数、点击行为；库内对话样式不变。

### 正常展示（S）

| 项 | 规则 |
|----|------|
| **工作区 chip 文案** | **`资料库名 · 文档名 · 章节 · p.页码`**（库名在最前） |
| **示例** | `答辩演示库 · golden_handbook.md · 1.1 年假` |
| **条数上限** | 最多 **5 条**（与现网 Top-K 一致） |
| **顺序** | 相关度从高到低（rerank 后） |
| **点 chip** | 打开 **该引用所属库** 的文档预览 |
| **答完自动展开** | 仍展开 **第 1 条** 摘录（R4-3） |
| **库内对话** | **不变**：`文档名 · 章节 · p.页码`（不写库名） |
| **单库命中** | 工作区对话 **仍带库名**（让用户看见系统选的库） |

### 乱操作 / 边界（E）

| 乱操作 | 系统怎么处理 | 你怎么验 |
|--------|--------------|----------|
| E12 5 条来自 5 库 | 5 chip 库名各不同 | 多库同主题 |
| E13 同库两条 | 库名可重复 | 同库多章节 |
| E14 grant 撤销 / 删文档 | chip 灰态或「来源不可用」（ORG-1.7） | 撤 grant 后看历史 |
| E15 库名超长 | 截断 + 悬停全名 | 超长库名 |
| E16 只命中 1 库 | chip 仍带库名 | 侧栏对话单库问 |

### 用户确认记录

- **2026-07-08**：工作区引用 **一律带库名**；库内对话格式不变。

## G-1-4 权限 · 角色差异 ✅ 已确认（2026-07-08）

**这节定什么**：`/ask` 谁能用、检索范围与 ORG OrgScope 一致；不新增越权后门。

### 个人空间

| 谁 | `/ask` | 检索范围 |
|----|--------|----------|
| 个人用户 | ✅ | **仅个人库** |

### 团队空间

| 谁 | `/ask` | 检索范围 |
|----|--------|----------|
| Owner / 公司 Admin | ✅ | 当前部门上下文可见库；可选 **全公司** scope |
| 部门 Member | ✅ | 当前部门可见库（子树 + 公共 + grant） |
| **未分配**（无部门） | ❌ | Banner + **输入禁用**（对齐 ORG-2.5） |

### 其它规则

| 规则 | 说明 |
|------|------|
| 与库内对话权限 | `/ask` **不扩大** 可见库 |
| 限流 | 与库内 chat **共用** M5 · 30 次/小时/用户 |
| Member 写权限 | 不变（仍不能上传/删库） |

### 乱操作 / 边界（E）

| 乱操作 | 系统怎么处理 | 你怎么验 |
|--------|--------------|----------|
| E17 未分配硬闯 `/ask` | 禁用提问或跳概览 + Banner | 未指派 invite |
| E18 切部门后继续问 | 下一问按新部门 scope | 切部门 |
| E19 Admin 全公司 scope | 检索全公司可读库 | Admin + all |
| E20 个人空间问团队内容 | 搜不到 → 拒答 | 个人空间 |

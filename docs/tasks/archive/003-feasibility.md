## 3. 架构调整（相对 TECH-1 原案）

| 原设计 | 调整建议 | 原因 |
|--------|----------|------|
| Qdrant 独立服务 | **优先 pgvector（向量在 PostgreSQL）** | 2G 少跑一个吃内存容器 |
| Celery + Redis | **MVP：FastAPI BackgroundTasks**；P1 再 Redis | 减组件、减运维 |
| 全 Docker 本机开发 | **本机 venv 开发 + 云上精简 Compose** | Docker 初学；降低上手门槛 |
| 对话 UI i18n | **P1** | 用户不会 React，MVP 界面中文 |
| 对话回答语言 | **MVP**：随用户提问中英 | Prompt 即可 |
| OCR / 扫描 PDF | **Wave 2** | 不确定 PDF 类型，主 demo 选手文字版 |

### 2G 云「演示版」目标形态

```
Nginx → FastAPI + 静态前端
PostgreSQL（含 pgvector）
（无 Qdrant / 无 Redis / 无 Celery）
```

本机开发可仍用 Docker 起 Postgres；与云上一致即可。

## 4. UI 风格参考（用户提供的三张图）

| **AURORA** | 白底、海军蓝主色、企业空间编号 → 登录 | **注册/登录/企业空间入口** |

**推荐组合（待用户敲定）**：

- 登录/企业：`AURORA` 结构 + 可选 `智联` 级背景图
- 登录后主应用：`蓝企鹅` 侧栏 + 卡片式知识库

**与 Signal 关系**：技术栈可借鉴目录/部署；**视觉必须独立**（配色、布局、字体不同）。

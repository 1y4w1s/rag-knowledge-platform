## 全文口播（约 480 字 · 正常语速 ≈ 2:00）

**【0:00～0:15 开场】**  
各位老师好。我的题目是《基于 RAG 的知岸系统设计与实现》。知岸是一款企业级知识库问答产品：团队上传 PDF、Word 等文档，AI 回答必须带引用，能点到原文页码，没有依据就明确说「未找到」，不胡编。

**【0:15～0:45 入库数据流】**  
系统有两条核心数据流。第一条是**入库**：管理员在网页上传文件，后端先校验格式和大小，存到按知识库隔离的目录，同时在 PostgreSQL 记一条文档记录。然后交给 FastAPI 的 BackgroundTasks 异步处理——解析正文、按章节结构切片、调用通义嵌入模型向量化，最后写入 pgvector 和业务表，状态变成「完成」。Dashboard 上能看到切片数和耗时。

**【0:45～1:15 问答数据流】**  
第二条是**问答**：用户提问后，先把问题嵌入，再在**当前知识库**里做 hybrid 检索——向量看语义相似，全文检索看关键词，RRF 融合后再 rerank，全程强制带 `kb_id` 过滤，防止查到别的库。检索到的片段和用户问题分开拼进 Prompt，DeepSeek 流式生成答案；引用块先于正文返回，含文档名、章节和页码，用户可点进预览核对。如果相关度不够，固定话术拒答，不吐假引用。

**【1:15～1:50 安全七层】**  
安全我按七层设计。**第一层** JWT 身份认证，密码 bcrypt。**第二层** RBAC：企业 admin 能管库和成员，member 只读上传禁、可对话。**第三层** 多租户隔离：所有 SQL 和向量检索都带 kb_id / workspace，SA-1～3 自动化测试防越权。**第四层** 上传白名单、MIME 双检、20MB 上限、内容指纹去重。**第五层** RAG 特有：防提示注入、检索与用户指令分离、无引用不胡编。**第六层** 审计日志记录登录、删库、上传等关键操作，外加登录和 API 限流。**第七层** 基础设施：LLM Key 仅服务端、Docker 内网 HTTP 部署、数据库不对公网暴露。

**【1:50～2:00 收尾】**  
目前 pytest 273 全绿，golden 检索 Hit@3 十二题全过，生产 smoke 已验。以上是我的系统数据流与安全设计，谢谢各位老师。

## 分层背诵卡片（每层一句）

| 层 | 你嘴里怎么说 | 评委可能追问 |
|----|-------------|-------------|
| 1 身份 | 登录发 JWT，密码 bcrypt 存库 | 为什么不用 session？→ 前后端分离、无状态好扩展 |
| 2 授权 | 每个 API 先验 token 再验对 kb 有没有权 | member 能删吗？→ 不能，403 + 前端 disabled |
| 3 隔离 | 向量检索强制 kb_id，A 的 token 访问 B 的库 → 403 | 怎么测？→ SA-1～3 pytest |
| 4 上传 | 白名单四类、20MB、SHA 去重 | 恶意 PDF？→ MVP 不做病毒扫描，P1 可选 ClamAV |
| 5 RAG | 片段和问题分块进 Prompt；没依据就拒答 | 幻觉怎么办？→ 相关性 gate + 必须带 citation |
| 6 审计 | 谁何时删库/上传有记录；登录 5 次/15min 锁 | 能看成员聊天吗？→ 默认不能，只记次数 |
| 7 基建 | Key 在 .env；Postgres 只在内网；内网 HTTP | 为什么没 HTTPS？→ 企业内网/VPN 场景，Wave 2 可加 TLS |

## 30 秒电梯版（备用）

知岸把企业文档做成可问答知识库。上传后异步切片入 pgvector；提问时 hybrid 检索加 rerank，答案必须带页码引用。安全上 JWT、RBAC、kb 隔离、上传校验、防注入拒答、审计限流、密钥不进前端，pytest 和 golden 评测闭环可证明检索质量。

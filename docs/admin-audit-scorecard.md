## 六维打分

| 维度 | 权重 | 得分 | 说明 |
|---|---|---|---|
| 一致性 Consistency | 20% | **9.4** | 统一外壳（active=操作审计，SVG 图标）/ 自定义 select 与全站同源 / 分页器复用 KBs / 中性动作标签符合「状态语义不可破」；无 token 漂移 |
| 可用性 Usability | 20% | **9.2** | 操作类型自定义下拉（键盘可达）、查询/重置/刷新均生效、分页器可翻页+跳转、三态切换无死控件；原生 date 输入可用 |
| 功能保全 Function | 20% | **10** | 五列（时间/操作/资料库/操作者/详情）与源码一致；18 类动作、缩短 UUID、详情截断、空/加载/失败三态、刷新、分页全部忠实；IP/资源列源码本就未展示，未擅加 |
| 无障碍 A11y | 15% | **8.9** | listbox+aria-expanded/aria-selected、th scope=col、输入均有 label、焦点环、按钮显式 type、reduced-motion 守卫 |
| 视觉 Visual | 15% | **9.0** | 层次清晰、节奏统一；中性动作标签消除原稿「一片琥珀」的视觉噪声，失败动作红色点睛 |
| 性能 Perf | 10% | **9.0** | 轻量、无重动效；骨架脉冲尊重 reduced-motion；背景漂移仅 will-change |

**加权总分** = 9.4×.2 + 9.2×.2 + 10×.2 + 8.9×.15 + 9.0×.15 + 9.0×.1
= 1.88 + 1.84 + 2.00 + 1.335 + 1.35 + 0.90 = **9.305 ≈ 9.31**

## 本轮关键改造（AA-R1）

| 编号 | 严重度 | 痛点 | 改造动作 |
|---|---|---|---|
| AA-R1-1 | 🟠 P1 | 源码用 `doc-badge-wait`（琥珀「处理中」）渲染**所有**审计动作，但审计条目是已发生事实、非进行中，违反「状态语义不可破 / 评审稿不撒谎」 | 改为**中性 label 标签**；对两类真正失败动作（登录失败 / 磁盘清理失败）用**红**标签（失败=红，语义诚实） |

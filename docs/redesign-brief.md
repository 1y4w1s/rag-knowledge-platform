# 屿知 · 全站推翻重做设计纲领（Redesign Design Brief）

> 定位：以专业前端设计师视角，先给现状 UI/UX 打分，再给出「完全推翻重做」的整体设计思路。
> 唯一硬约束：**保留现有主题色（赤陶橙 #CB6B3D + 暖白纸背）**，其余（页面、组件、交互、Logo、名字）全部可重新设计。
> 参考灵感：unicornui / motionsites / designspells / shadergradient / figcomponents / godly / variant。

---

## Part 1 — 现状 UI/UX 评分：**6.8 / 10**（能用、专业，但"模板感"重，缺记忆点）

| 维度 | 分 | 扣分原因（简要） |
|------|----|------------------|
| 品牌辨识度 | 6.0 | 赤陶橙是好资产，但只当"强调色"点缀；整体仍是 shadcn 默认布局的影子，遮住 Logo 换个色就像任何一个企业 SaaS，**没有让人记住的视觉符号**。 |
| 视觉层级 | 6.5 | 企业表格 + 卡片密度偏高，一屏信息过载；标题/正文/次要信息的对比不够拉开，重点不跳。 |
| 版式与留白 | 6.5 | 内容区节奏偏"挤"，缺少呼吸感和大留白；栅格不够克制，卡片大小不一致处较多。 |
| 动效与愉悦感 | 6.0 | 改造前几乎只有加载态；缺少滚动叙事、微交互、状态转场——对比灵感站的"活"，显得静。 |
| 一致性 | 6.0 | 令牌存在 v6/v7/v8 多版本并存（drift 风险），阴影/圆角/间距有多套写法，长期维护会越来越乱。 |
| 字体表现 | 7.5 | 衬线做标题的"编辑气质"是亮点、方向对；但正文 sans 偏通用，字阶未系统化。 |
| 交互反馈 | 7.5 | 空态/错态/权限拒绝做得比多数项目好（企业向加分）；但反馈样式不统一。 |
| 可访问性 | 7.5 | 焦点态、reduced-motion 有照顾，基础扎实。 |

**一句话诊断**：不是"丑"，是"没有个性 + 不够精致"。骨架是合格的企业产品，但缺一套**强观点的设计语言**把赤陶橙从"点缀"变成"灵魂"，也缺灵感站那种"每个细节都被设计过"的手感。所以推翻重做的价值在于——**建立记忆点 + 系统化 + 注入克制的高级动效**，而不是推翻配色。

## Part 2 — 推翻重做整体设计思路

### 2.1 视觉风格定位：**"暖科技编辑感"（Warm Editorial Tech）**

一句话风格锚：**像一本被精心排版的出版物，又像一个可靠的企业级工具。**

- **反面清单**（明确不做）：不做暗色玻璃拟态全站、不做霓虹/赛博、不做 shadcn 默认冷灰、不做 AI 模板式"万能居中空态 + emoji 插画"。
- **正面锚点**：
  - 从 **variant / designspells** 取"编辑感排版"——大留白、强字体层级、内容即主角。
  - 从 **shadergradient / motionsites** 取"暖色流动光"——但只用在 Hero/登录等少数英雄区，做成赤陶色的极慢呼吸光晕（20s+），不铺满全站。
  - 从 **godly / figcomponents** 取"微交互精致度"——按钮扫光、卡片景深、滚动错峰入场、hover 有反馈。
- **气质关键词**：温暖、可信、克制、有出版物的秩序感。

### 2.2 品牌重塑：新名字 + 新 Logo

现名"知岸"意象好（知识靠岸），但重做可顺势升级识别度。围绕"从信息汪洋抵达确定引用之岸"的核心叙事：

| 候选名 | 含义 | 气质 |
|--------|------|------|
| **屿知（YUZHI）** ⭐推荐 | 知识之屿，信息海中可停靠的小岛 | 温暖、独特、易记、可国际化 |
| 砚台（YAN） | 研墨治学，东方编辑感 | 文气重，偏内容/研究向 |
| 壤 / Loam | 沃土，知识生长的基质 | 抽象、高级，但需更多解释 |

- **Logo 概念**：赤陶圆角方形 tile 内嵌"**地平线弧 + 灯塔竖笔 + 顶点光**"——既是"岸/屿"的抽象，也是"灯塔=指引/可信"的隐喻。单色可用、可缩成 16px favicon、暗底反白均成立（见设计板 01 区）。
- **Wordmark**：衬线中文字 + 宽字距拉丁副名，强化"出版物"气质。

> 名字/Logo 是建议项——你若想保留"知岸"，本纲领其余部分完全不受影响。

### 2.3 色彩体系：把赤陶橙从"强调色"升级为"完整主色阶"

这是重做的核心。现状把 #CB6B3D 当单点强调，重做要建**7 级明暗阶 + 暖中性 + 语义色**（见设计板 02 区）：

- **主色阶 Terracotta 50→900**：`#FBEEE4 / #F3CDAE / #E8A778 / #CB6B3D(400 主) / #A8451E / #6F2710 / #4A1B0C`
  - 50/100 做浅底与选中态，400 做主 CTA/链接，600/800 做文字与深色区、Hero 渐变。
- **暖中性**：纸背 `#FAFAF8`、面 `#F5F0EB`、描边 `#EFEAE4`、次文字 `#71717A`、主文字 `#18181B`——全部微微偏暖，杜绝冷灰。
- **语义色**：成功绿 / 处理中琥珀 / 错误砖红，均向暖调统一，**不用系统冷灰红**，让"引用失效/权限拒绝"等企业错态也有品牌温度。
- **令牌治理**：废弃 v6/v7/v8 并存，收敛为**单一版本 token 源**（`--clay-50…900 / --warm-* / --semantic-*`），Tailwind 主题从 token 生成，杜绝 drift。

### 2.4 字体与字阶：衬线担纲"编辑气质"

- **Display / Heading**：衬线（Noto Serif SC / Songti），用于 Hero 主标题、页面标题、卡片分区标题——这是品牌"编辑感"的核心载体。
- **Body / UI**：无衬线（系统栈），正文、表单、导航。
- **Data / Metric**：等宽（tabular-nums），所有指标、命中率、耗时、计数——强化"数据可信"感。
- **系统化字阶**：Display 30/36 · H1 24/32 · H2 18/26 · Body 13-14/1.6 · Caption 12 · Mono 12-13。两种字重为主（Regular / Medium），克制。

### 2.5 页面布局结构：三段式壳 + 内容即主角

- **全局壳（App Shell）**：
  - 左侧 **导航侧栏**（毛玻璃暖白 + 主色选中态）——只放导航，不塞筛选。
  - 顶部 **上下文栏**（面包屑 + 全局搜索 + 工作区切换 + 头像）——导航语义，筛选/搜索下沉到内容区。
  - 右侧 **内容区**：`max-width ~1080px` 居中，大留白，8pt 栅格。
- **关键页蓝图**：
  - **登录/落地**：全屏英雄区 + 赤陶呼吸光晕 + 衬线大标题，第一印象即"高级"。
  - **概览 Dashboard**：Hero 问候卡（品牌渐变 + 光晕 + 数字递增）→ KPI 卡组 → 最近资料库 → 检索质量指标，滚动错峰入场。
  - **资料库列表/详情**：卡片网格，hover 景深 + 缩略图微动；详情页左文档树右预览，引用可点。
  - **对话页**：固定输入框（不随滚动下沉）、消息流带**引用溯源卡片**（文档名+位置+片段可展开）——这是产品 P0，视觉上要突出。
  - **组织/设置**：表单克制、分区清晰、权限态可理解。

### 2.6 信息层级与导航设计

- **三级信息层级**：① 页面主标题（衬线大字）② 分区标题 + 操作入口 ③ 卡片/行内内容。用字号、字重、留白三者拉开，而非靠边框堆叠。
- **导航原则**：顶栏=去哪里（导航），内容区顶部=看什么（筛选/搜索/tab）。避免把 ⌘K、筛选塞进顶栏与壳冲突。
- **降噪**：企业表格默认收起次要列，hover/展开再现；一屏只给一个视觉重心。
- **可追责错态**：权限拒绝、引用失效、入库失败等，用语义色 + 明确文案 + 下一步动作，杜绝裸 `Not Found`。

### 2.7 关键交互体验（克制的高级动效）

全部尊重 `prefers-reduced-motion`，动效服务于理解而非炫技：

1. **英雄区呼吸光晕**：Hero/登录用赤陶色 20s 极慢流动光（shadergradient 暖色版），营造"活着"的高级感，仅少数英雄区使用。
2. **滚动错峰入场**：内容分区进入视口时上浮淡入（godly 招牌节奏），首屏之后渐次展开。
3. **数字递增**：KPI/指标从 0 缓动到位，强化数据感。
4. **主按钮扫光 + 微抬升**：brandGrad 主 CTA hover 时一道高光扫过 + 轻微上移。
5. **卡片景深**：资料库卡 hover 抬升 + 主色描边 + 缩略图微动。
6. **状态转场**：加载→内容用骨架屏平滑过渡；入库进度实时反馈。

### 2.8 组件库与设计系统落地

- **单一 token 源**：颜色/间距/圆角/阴影/动效缓动全部 token 化，Tailwind 主题从 token 生成。
- **基础组件重做**：Button（主/次/幽灵/危险，含扫光变体）、Card（含 top-accent + lift）、Input/Select（暖描边 + 焦点态）、Badge（语义色）、Table、Empty/Error/Loading 三态、Toast、Modal、Tabs、Nav。
- **动效原语**：`<Reveal>`（滚动入场）、`<CountUp>`（数字递增）、`hero-aurora` / `btn-shine`（CSS 工具类）——已在当前 v8 有雏形，重做时纳入正式组件层。
- **文档**：Storybook 或 design-preview 页承载所有组件的规范态，确保团队一致。

## Part 3 — 落地节奏（建议，非本次执行）

1. **W1 设计系统底座**：token 收敛 + 色阶/字阶/间距落地 + Button/Card/Input 三大基础件。
2. **W2 品牌壳**：新 Logo/名字（若采纳）+ App Shell（侧栏/顶栏）+ 登录英雄区。
3. **W3 核心页**：Dashboard → 资料库列表/详情 → 对话页（引用溯源卡片）。
4. **W4 长尾页 + 动效抛光**：组织/设置 + 滚动叙事 + 微交互 + 三态统一。
5. **全程**：`npm run build` 绿 + 关键页浏览器验收；不破坏后端契约与权限隔离。

## 1. 这脚本验收什么（30 秒）

公司空间里，不同部门的人 **API 层** 就看不到兄弟部门的库——不是页面藏行。Admin 建部门树、把人挂进研发部；研发成员 **看不见** 市场机密库；人事部的「员工手册」经 **跨部门共享（grant）** 全公司只读后，研发成员才能对话引用；撤销 grant 后立刻失效。

**主线四段**：建树与挂人 → 研发/市场隔离 → 人事部库 + 保密库 → grant 开放与撤销。

## 3. 全链路操作表（15 步 · 主流程）

> **用法**：从左到右跟着点；「预期」列打 ✅ 才算该步过关。  
> **角色切换**：步骤 5～7、10、12～13、15 用 `demo_member`；步骤 1～4、8～9、11、14 用 `demo_admin`。每次从 member 切回 admin（步骤 8、11、14 前）须 **退出登录** 再登 `demo_admin`。

| # | 账号 | 操作（页面路径） | 预期（看见什么） | PRD 对照 |
|---|------|------------------|------------------|----------|
| **1** | admin | 登录 → 侧栏 **组织与部门** → 点 **+ 新建一级部门** 三次，分别命名 **研发部**、**市场部**、**人事部** | 左树在根节点「知岸演示公司」下出现 3 个一级节点 | ORG-1-3 S1 |
| **2** | admin | 左树点 **研发部** → 右栏 **添加成员** → 选 `demo-member@example.com` · 部门角色 **部门成员** · 勾选 **设为主部门** → **添加** | 成员表有演示成员 · 主部门标记 | ORG-1-3 S3 |
| **3** | admin | 侧栏 segmented 选 **知岸演示公司**（团队空间）→ **当前部门 ▾** 选 **研发部** → **知识库** → **新建资料库**「研发内部库」· 归属 **当前部门** | 列表出现「研发内部库」 | ORG-1-4 S1 · 研发 |
| **4** | admin | **当前部门 ▾** 切 **市场部** → **新建资料库**「市场机密库」· 归属 **当前部门** → 自动进详情后 **记下地址栏** `/knowledge-bases/{id}` | 列表见「市场机密库」（及已有公共库若有）· **不见**研发库（当前 scope）· URL 已抄下供步骤 7 | 市场 |
| **5** | member | **退出登录** → `demo_member` / `password123` 登录 → 进团队空间 | 侧栏 **当前部门** 显示 **研发部** · Popover **无**「市场部」「全公司」 | ORG-1-2 S1 |
| **6** | member | **知识库** 列表 | 有 **研发内部库** · **无**「市场机密库」 | ORG-1-1 S3 · 研发隔离 |
| **7** | member | 用 admin 在步骤 4 记下的市场库 URL（`/knowledge-bases/{id}`）**地址栏硬闯** | Toast **「该资源不在当前工作区」** · 跳 **概览** · 进不了详情 | ORG-1-1 E1 |
| **8** | admin | **退出登录** → `demo_admin` 登录 → **当前部门** 选 **人事部** → 新建「**员工手册（人事）**」· 归属 **当前部门** → 进详情 **上传** `golden_handbook.md` · 等 **完成** | 文档状态 completed | 人事 |
| **9** | admin | 同部门再建「**薪酬保密库**」· 归属人事部 · **不上传** · **不**开共享 | 仅 admin/人事上下文可见 | ORG-1-1 S5 |
| **10** | member | 刷新 **知识库** 列表 | **无**「员工手册（人事）」· **无**「薪酬保密库」 | 人事隔离 |
| **11** | admin | **退出登录** → `demo_admin` 登录 → 进「员工手册（人事）」详情 → 展开 **跨部门共享** → 目标 **全公司** · 权限 **只读** → **添加共享** | Toast「已添加共享」· 列表出现「全公司 · 只读」 | ORG-1-4 S3 · grant |
| **12** | member | 刷新 **知识库** 列表 | **出现**「员工手册（人事）」· 仍 **无**「薪酬保密库」 | grant 生效 |
| **13** | member | 进「员工手册（人事）」→ **开始对话** → 问：**「年假有多少天？」** | 流式回答 · 下方 **引用 chip**（手册章节） | ORG-1-4 S3 · 对话 |
| **14** | admin | **退出登录** → `demo_admin` 登录 → 同一库 **跨部门共享** → 点该 grant 的 **撤销** | Toast「已撤销共享」· grant 行消失 | ORG-1-4 S4 |
| **15** | member | 刷新列表 → 再 **开始对话** 或旧 thread 问手册内容 | 列表 **无**该库 · 对话 **不**再命中手册（无相关引用） | grant 撤销 |

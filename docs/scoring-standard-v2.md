# 评分标准 v2（提高版门禁）

> 背景：10 个页面已全部通过 v1 门禁（六维均 ≥ 8.0，均值 ≈ 9.29）。本文件在 v1 基础上**提高标准**，用于「返工审计」阶段。

## 1. v1 → v2 变化

| 项 | v1（已达标） | v2（提高后） |
|---|---|---|
| 六维下限 | 每维 ≥ 8.0 | **每维 ≥ 9.0** |
| 加权下限 | — | **加权 ≥ 9.2** |
| 跨页一致性 | 建议项 | **硬性地板（任一违反即判不达标）** |

> 权重不变：一致性 20% · 可用性 20% · 功能 20% · 无障碍 15% · 视觉 15% · 性能 10%。

## 2. 跨页一致性地板（硬规则）

满足以下任一条即视为**不达标**，须返工：

1. **图标/控件一律 SVG**：统一外壳的导航图标、工作区切换 caret、下拉箭头等，禁止字符型 caret（`▾`/`▸`）或 emoji 图标。线性 SVG 规范：18px、`stroke:currentColor`、线宽 1.6、圆角端点。
2. **状态色专用**：绿/琥珀/红仅用于真实状态语义（完成/进行中/失败）。角色、标签、分类等非状态元素**不得**挪用状态色；角色徽章须用独立专用色。
3. **原生控件禁止**：原生 `<select>` / `<input type=radio>` 一律替换为自定义控件（listbox / check）。
4. **无死控件**：所有可见按钮/链接须真实可达或事件生效；评审稿不撒谎。

## 3. v2 逐维 9.0+ 评审要点

- **一致性 9.0+**：与 design-system token 完全一致；跨页控件（按钮/徽章/表格/分页/对话框）同源；无冷色漂移。
- **可用性 9.0+**：主要操作一键可达；反馈明确；加载/失败/空态齐备；无歧义。
- **功能保全 9.0+**：原功能零增零减（10 为上限，非状态页一般 9.5–10）。
- **无障碍 9.0+**：语义角色齐全（`nav`/`main`/`th[scope]`/`role=dialog`/`aria-*`）；所有按钮显式 `type`；焦点 `:focus-visible` 赤陶环；键盘可达。
- **视觉 9.0+**：层次清晰、留白有节奏、视觉语言统一；主区/卡片对比度达标。
- **性能 9.0+**：纯静态无重资源；骨架/`prefers-reduced-motion` 守卫；长列表虚拟化（如适用）。

## 4. 应用方式

- 逐页用本文件 + `rework-audit.md` 核对；
- 仅违反「跨页一致性地板」的，做定向返工（如 caret→SVG、角色专用色）；
- 维度分 < 9.0 的，做针对性提升或重新评估（说明依据）；
- 全部达标后进入「同步回代码」阶段。

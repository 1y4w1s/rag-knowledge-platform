# 知岸 · 全页重设计评分卡（Scoring Session）

> **评分标尺**：以 `design-system.md`（统一 token/组件）与 `app-shell-audit.md`（外壳规范）为基准，对四张已出预览的页面逐项打分。
> **维度与权重**：一致性 20% · 可用性 20% · 功能保全 20% · 无障碍 15% · 视觉 15% · 性能 10%。
> **分值**：0–10（10=标杆）。预览稿默认保留「原版功能零增零减」，故功能保全普遍满分；扣分项集中在跨页一致性 / 外壳 / 已知限制。
> **被测页面**：Login（登录）· KnowledgeBases（资料库列表）· KnowledgeBaseDetail（资料库详情）· Chat（对话）。

## 2. 逐页点评

### 2.1 登录 Login · 9.05
- ✅ **亮点**：遵守「仅改背景、卡片原版」锁规则；暖白中等渐变 + 赤陶点缀，视觉疲劳最低；功能 100% 保留。
- ⚠ **短板**：无障碍 8——表单仍依赖原生控件语义，未做聚焦环外显（登录域优先级低于对话/详情，暂缓）。
- 💡 建议：可补 `@media reduce` 守卫（已全局有），即可升至 9.2。

### 2.2 资料库列表 KnowledgeBases · 8.50
- ✅ **亮点**：严格零新增功能，仅修状态色（绿/琥珀/红）+ 状态文字徽章 + 三按钮固定底部；超长名/描述 edge-case 已覆盖。
- ⚠ **短板**：一致性/视觉 8——卡片密度与详情页表格节奏未完全对齐；列表项 hover 态弱。
- 💡 建议：补卡片 hover 浅暖底 + 与详情页同款阴影，可升至 8.8。

### 2.3 资料库详情 KnowledgeBaseDetail · 9.10（当前最高）
- ✅ **亮点**：经过 18 项修复 + Round 1 迭代；状态语义统一、自定义下拉替原生、表格真正圆角、共享面板默认收起、`<th scope>` 完整、背景层 `will-change` 提升。
- ⚠ **短板**：性能 8——大列表（>1000 篇）未评估虚拟滚动（真实代码待办，预览规模小无虞）。
- 💡 建议：同步回代码时评估 windowing，可满分。

### 2.4 对话 Chat · 8.60（本轮新建）
- ✅ **亮点**：忠实还原 `ChatPage` 组件树（侧栏壳 + 线程面板 + 工具栏 + 模式切换 + 用户/助手气泡 + 引用 chip + 审批卡 + 空/加载态）；原生 KB `select` 已替换为自定义下拉；流式光标/检索态/日期分隔齐备；移动抽屉可交互。
- ⚠ **短板**：
  - 一致性 8——**外壳透明度不统一**（侧栏 `.72` vs 顶栏 `.55`，见 S1）、**内容 `-m-6` breakout**（S2）尚未在预览内修正（预览为静态呈现，按规范 §3 思路搭，但代码层 S1/S2 待同步）。
  - 无障碍 8——引用 chip 展开为 click-only；线程删除仅有 title 无可见 label（代码层有 aria-label，预览保留）。
- 💡 建议：落地 `app-shell-audit.md` 的「统一 `--c-shell` + `full-bleed`」后，一致性可升至 9，总分约 8.9。

## 4. 总评与下一步

**总评**：设计语言已收敛为单一可复用系统，功能零损失，无 P0 缺陷。距「标杆 9.5+」差两块拼图：
1. **外壳统一**（S1/S2）——同步 `AppTopbar`/`ChatPage` 到 `--c-shell` + `full-bleed`，消除透明度/内距错位。
2. **字体限制**（H5）——中文衬线在 Windows 偏细，若登录/品牌字标要更强可读性，可整体改无衬线加粗（需评估与全站一致性）。

**下一步候选（按优先级）**：
1. 同步回代码：先详情页（台账最完整）→ Chat（含 S1/S2）→ KBs → Login。
2. 继续下一页：AskPage（与 Chat 同域，需核对视觉同源，见 S4）。
3. 其余页：AccountSettings / Members / OrgDepartments / OrgSettings / AdminAudit / DocumentPreview。

> 评分环节结束。本卡与 `design-system.md`、`app-shell-audit.md`、`knowledge-base-detail-design-review.md` 共同构成「预览 ↔ 代码」同步的事实来源。

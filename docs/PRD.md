## 1. 一句话定位

**睿阁**是一个支持多格式文档上传、基于知识库 AI 对话、并标注引用来源的智能问答系统——面向个人学习/自用，以及毕业设计演示；账号分为**个人版**与**企业版**两种类型。

| 项 | 内容 |
|----|------|
| **中文产品名** | **睿阁** |
| **代码仓库目录** | `rag-knowledge-platform`（暂不改文件夹名，避免牵一发动全身） |
| **答辩可用全称** | **基于 RAG 的知岸系统设计与实现** |

## 2. 目标用户与使用场景

| 用户 | 场景 |
|------|------|
| **你本人** | 日常整理 PDF/文档，对知识库提问，验证 RAG 效果 |
| **毕业设计答辩** | 演示：上传文档 → 创建知识库 → AI 对话 → 展示引用溯源 |
| **个人版用户** | 单人使用，管理自己的知识库与对话 |
| **企业版用户** | 以「组织」为单位，管理员邀请成员，共享组织内知识库 |

### 2.1 引用来源标注（P0 硬性要求）

**每条 AI 回复必须带引用来源**，不是可选项。具体包括：

| 要求 | 说明 |
|------|------|
| **必须有** | 每条 AI 回答下方展示引用列表 |
| **引用内容** | 来源文档名 + 页码或段落位置 + 可展开原文片段 |
| **无依据时** | 明确说明「知识库中未找到相关内容」，**禁止无引用胡编** |
| **答辩演示** | 必须能指着引用向老师证明「答案来自哪份文档」 |

## 3. 个人版 vs 企业版（核心差异）

> **2026-07-05 增补（WS-2）**：账号仍有 `account_type`（个人 / 企业注册路径），但**登录后权限与数据范围以「当前工作区」为准**——**我的空间**（个人资料库）与 **团队空间**（组织资料库）可并存、可切换。壳层与 API 细节见 `docs/tasks/workspace-prd-ws2.md` **WS-2-1～WS-2-3**。

| 维度 | 我的空间（personal workspace） | 团队空间（team workspace） |
|------|-------------------------------|----------------------------|
| **谁有** | 所有登录用户 | 已加入某组织的用户（含个人账号凭邀请码加入者） |
| **资料库归属** | `owner_user_id = 我` 的个人库 | `owner_org_id = 当前组织` 的团队库 |
| **列表/统计 scope** | 仅个人库及其文档/对话 | 仅当前组织团队库及下属数据；**禁止**同一页混排个人+团队 |
| **成员/花名册** | ❌ 无成员入口 | **Admin**：成员管理（邀请/移除/改角色）· **Member**：**只读**团队成员花名册（见 §5.8） |
| **团队设置** | ❌ 隐藏 | ✅ 仅 **Admin**（侧栏「团队设置」） |
| **写权限（建库/上传/删库/删文档）** | 本人 | **Admin / Owner**；Member **只读 + 可对话** |
| **计费/付费** | ❌ MVP 不做 | ❌ MVP 不做（未来见 §14） |

**与 `account_type` 的关系（答辩必讲清）**：

- **注册**仍分「个人 / 团队」路径（团队含「创建团队」或「邀请码加入」）；不等于「一辈子只能待在一个空间」。
- **个人账号**加入团队后：侧栏出现 **我的空间 ↔ 团队** 切换器；在「我的空间」仍只有个人库，在「团队空间」按 `org_role` 行使权限。
- **权限判据**：**不再**用 `account_type === enterprise` 决定侧栏与管理能力；改为 **`当前工作区 === 团队 && org_role === admin`**（Owner 视为 admin，见 WS-1-6）。

### 3.1 团队空间角色权限（WS-2 / WS-1-6 定稿）

| 角色 | 能做什么 | 不能做什么 |
|------|----------|------------|
| **组织管理员 Admin** | 邀请/移除成员（不含 Owner）；创建/删除团队资料库；上传/删文档；对话；改组织展示名 | 不能移除/降级 **Owner**；改角色仅 Owner 可操作 |
| **组织所有者 Owner** | 创建团队的用户；权限同 Admin + **唯一**可设/降 Admin、转让 Owner | 不能被其他 Admin 移除或降级；离开前须转让 |
| **成员 Member** | 查看团队资料库列表；对已授权资料库 **对话**；只读查看文档列表；**查看团队成员花名册（只读）**——邮箱 · 昵称 · 角色 · 加入时间 | 邀请/移除/改角色；删库；上传/删文档；组织设置；邀请码管理 |

**MVP 简化（与 Q1 一致）**：

- 团队 **Member 默认只读 + 可对话 + 可看花名册**；上传/删库仅 Admin/Owner。
- 成员 **不能**上传文档（Q1 ✅）；若未来开「编辑者」角色再扩表。

### 3.2 工作区切换（WS-2-1 摘要）

| 行为 | 定稿 |
|------|------|
| 切换器位置 | 侧栏 Brand 下方 segmented：**我的空间** / **{组织名}** |
| 切换后 | 跳转 **`/dashboard`**；统计与列表随 workspace 重拉 |
| API | 请求带 `workspace=personal` 或 `workspace={org_id}` |
| 持久化 | `localStorage` 记住上次工作区；与 `/me` 对齐，脏数据回落 personal |

## 4. 核心用户路径

### 4.1 个人用户主路径

```
注册/登录（个人） → 创建知识库 → 上传文档 → 等待入库完成 → 进入对话页 → 提问 → 看 AI 回答 + 引用来源
```

### 4.2 企业用户主路径

```
注册/登录（企业 + 组织名） → 创建组织知识库 → 上传文档 → 邀请成员 → 成员登录 → 选择知识库对话 → 看回答 + 引用
```

### 4.3 文档入库路径（后台，用户可感知进度）

```
上传文件 → 显示「处理中」→ 解析 → 切片 → 向量化 → 状态变「已完成」/「失败（可重试）」
```

### 5.1 登录 / 注册页

> **WS-1-2**：注册为 **三步向导**（用法个人/团队 → 团队时创建者/邀请码 → 凭据），非单页「个人/企业」单选；个人账号可在账号设置 **凭邀请码加入团队**。

| 元素 | 行为 |
|------|------|
| Tab「登录」/「注册」 | 切换表单 |
| 注册 Step 1 | 「个人」/「团队」用法选择 |
| 注册 Step 2（团队） | 「创建团队」（填组织名）或「邀请码加入」 |
| 注册 Step 3 | 邮箱 + 用户名 + 密码 → 提交 |
| 登录 | 邮箱 + 密码 → **概览 Dashboard** |
| 「退出登录」 | 头像菜单 → 退出 → 回登录页 |

> **工作区 `/ask`** 多库检索见 `docs/tasks/discovery-smart-chat-prd.md` **G-1**；**多会话 thread** 见 `docs/tasks/discovery-smart-chat-g2-threads-prd.md` **G-2** · TECH **TECH-5.8**；**快速/精准两档** 见 `docs/tasks/discovery-agent-g3-read-prd.md` **G-3** · TECH **TECH-7**。

| 元素 | 行为 |

## 14. 未来扩展：积分与付费（Wave 2+，MVP 不做）

> **决策（2026-07-03）**：用户确认「先完成基础功能」；本节仅存档方向，**不影响 MVP 范围与开发清单**。

### 14.1 为什么要积分制

睿阁每次对话/入库都消耗 LLM API。积分 = 用户可见的「额度」，便于统一 DeepSeek / 通义等不同模型的扣费。

### 14.2 规划能力（答辩后再做）

| 能力 | 说明 |
|------|------|
| 钱包 + 流水 | 余额；充值/消费/赠送/退款记录 |
| 对话扣积分 | 按 token 用量折算扣减 |
| 入库扣积分 | 按页数/文件大小（解析+嵌入） |
| 个人版 | 积分挂用户账号 |
| 企业版 | 积分挂组织；管理员充值，成员共用 |
| 真支付 | 微信/支付宝回调（需商户资质，Wave 2 最晚） |

### 14.3 建议落地顺序

1. **Wave 2a**：注册送积分 + 用完不能问（不接支付，可演示额度）
2. **Wave 2b**：模拟充值按钮（假支付，毕设故事用）
3. **Wave 2c**：接真支付

### 14.4 MVP 预留（Implement 时可记，现在不建表）

- 用户表留 `plan_type` 或类似字段即可；**MVP 不实现扣费逻辑**。

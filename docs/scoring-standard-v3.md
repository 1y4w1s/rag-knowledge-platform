# 评分标准 v3 · 极致严苛评审体系

> 目的：在 v2 基础上**近乎疯魔级别**地收紧所有维度，覆盖代码质量 / 性能 / 可访问性 / 安全性 / 用户体验 / 视觉 / 可维护性 / 国际化 / 响应式 / 可观测 / SEO / 一致性 12 维。**任一硬阈值未达 = 该维扣到 B 以下**；**任一一票否决 = 整页 D 档**。
>
> 权威文档。本文件替代 `scoring-standard-v2.md`，所有 `< 9.0` 的页面必须按本文件复评。
>
> 配套：UI 集成在 `previews-gallery.html`（评估中心 v3），违规编号 `H?-?` 全程贯穿。

## 0. 评分刻度（5 档）

| 档位 | 加权分 | 含义 | 处置 |
|---|---|---|---|
| **S 卓越** | ≥ 9.5 | 标杆级，行业领先 | 直接通过；可作为范本 |
| **A 优秀** | 9.0 ≤ x < 9.5 | 达标，细节可再打磨 | 通过；记"打磨项" |
| **B 良好** | 8.0 ≤ x < 9.0 | 局部短板，需返工 | **不通过**，需 ≥1 轮返工 |
| **C 待改进** | 6.0 ≤ x < 8.0 | 系统性缺陷 | **不通过**，需 ≥2 轮返工 |
| **D 不达标** | < 6.0 或触发一票否决 | 存在严重问题 | **驳回重做** |

**降级规则**（强于 v2）：
1. 加权 < 9.0 = 不通过（v2 同条款）
2. **任一一票否决 = D 档**（v2 没有）
3. **任一维 < 8.0 = 强制降档**（v2 没有）
4. 全部硬指标必须程序化测量，**未测 = 该项 0 分**（v2 没有）
5. **D5 视觉美学分项**主观比例 ≤ 30%（其余按可测规则）

## 2. 硬指标（H? = Hard 指标）

每条硬指标 = **必须程序化测量**。未测量项在该维 0 分。每条配扣分规则。

### D1 视觉一致性（12%）

| 编号 | 硬指标 | 扣分规则 |
|---|---|---|
| H1.1 | 主色板与 design-system token 100% 对齐（`--terracotta / --amber / --ok / --err / --role` 5 套色不可漂移） | 1 处漂移 -0.2，3 处 -0.5 |
| H1.2 | 字体尺度走 token（`font-size` 必须为 token 之一：`11/11.5/12/12.5/13/13.5/14/15.5/16/18/20/24`） | 1 个不命中 -0.1，5+ -0.4 |
| H1.3 | 圆角 4 级（`4/8/10/14/18`），不出现 `border-radius:7px` 等游离值 | 1 处游离 -0.1 |
| H1.4 | 间距 4 倍数（`4/6/8/10/12/14/16/18/20/24/32`），不出现 `13px` 之类 | 1 处游离 -0.1 |
| H1.5 | 图标 100% SVG（`stroke:currentColor`、`stroke-width:1.6`、`stroke-linecap/linejoin:round`） | 1 个非 SVG -0.5，3+ 一票否决 |
| H1.6 | 状态色专用（绿/琥珀/红**仅**用于完成/进行中/失败语义；角色、标签、分类不得挪用） | 1 处挪用 -0.5 |

**达标线**：≤ 1 个 H 指标违反且无严重偏移 → A 档；3+ → B 档。

## 3. 评审流程（每页 3 步）

```
A. 程序化扫描（5min）
   ├── H1.* / H5.4 / H6.4 / H7.9 / H8.* / H9.2 自动检查
   ├── 输出违规编号 + 计数
   └── 触发一票否决 = 直接 D 档

B. 视觉复核（10min）
   ├── H4.*（焦点环/对比度/键盘）
   ├── H5.1-5.3（节奏/对齐/留白）
   ├── 主观美学打分
   └── 截图存档

C. 综合评级
   ├── 加权计算（v3 权重表）
   ├── 降档检查（任一维 < 8.0 → 降档）
   └── 输出 S/A/B/C/D + 违规清单
```

## 4. 应用方式

1. 每页先走 §3 流程
2. 触发一票否决 → 直接 D，不走加权
3. 加权 ≥ 9.0 且无 < 8.0 维度 → A 通过
4. 加权 < 9.0 → B/C 不通过，进返工队列
5. 全部 A 档 → 「同步回代码」阶段

## 五、改进建议（优先级）

### P0 · 立即修复
1. **本地化后端错误文案**：在 `ChatMessageList` 渲染 assistant content 时，对后端返回的英文无结果/异常文案做映射，例如：
   - `"No relevant content was found..."` → `"未在资料库中找到相关内容，无法生成回答。"`
   - 或直接推动后端统一返回 i18n key，前端按 `lang` 渲染。

### P1 · 尽快优化
2. **移动端顶栏重构**：在 `ChatToolbar` 移动端将"当前资料库/当前空间"改为下拉菜单单独一行或折叠进菜单，避免与"引用溯源"、"新建对话"挤在同一行。
3. **键盘可达性**：
   - 添加 skip link（"跳到输入框"）
   - 或把历史列表从 Tab 序列中降级：线程项本身可点击，删除按钮仅在 hover/focus 时显示，减少 Tab 步数。
4. **清理临时产物**：移除或归档 `preview-chat-real.tsx`、`preview-empty-chat.tsx`、临时路由、以及 `docs/shot_*.py`、`docs/test_*.py` 中已完成的验证脚本。

### P2 · 体验打磨
5. **统一焦点样式**：把 `:focus-visible` 统一为设计令牌的陶土色 2px outline，替换浏览器默认黑框。
6. **补全 SEO 元数据**：为 `/ask`、 `/knowledge-bases/:id/chat` 加 description、canonical、og:title。
7. **模式激活态对比度**：将激活模式 pill 的文字加深或背景加深，确保正常 AA（≥4.5）。

## 六、与画廊预览的差距总结

画廊预览给 ChatPage 设定约 9.08、AskPage 约 9.22，当前真实实现 8.15。差距主要来自"真实运行后才会暴露"的工程问题，而非视觉风格：

- 英文错误文案把 D11 从 9.5 拉到 6.0，直接拉低总分约 0.14 分；
- 移动端顶栏与键盘可达性把 D9/D4/D2 各拉低 0.5–1.0 分；
- 临时文件与重复代码把 D10/D7 拉低约 1.0 分；
- SEO 缺失把 D12 拉低约 1.0 分。

**建议目标**：完成 P0+P1 后，可稳上 **8.8–9.0（A-）**；再完成 P2 打磨可达 **9.1+（A）**。

## 2. 本轮改动（1 项，D8）

| 文件 | 改动 | 维度 |
|---|---|---|
| `src/components/dashboard/SearchSnippet.tsx` | 移除 `dangerouslySetInnerHTML`，改为**零依赖安全解析器**：按 `<mark>` 切分，标签外纯文本交 React 转义、标签内渲染 `<mark>` 元素 | D8 |
| `src/components/dashboard/SearchSnippet.test.tsx`（新增） | 6 个用例证明安全：单/多高亮、无高亮、未闭合容错、嵌入 `<script>` 被转义为纯文本（无 XSS 面）、空输入返回 null | D7/D8 |

### 为什么比「受控注入」更安全
- 旧版用 `dangerouslySetInnerHTML` 注入 `ts_headline` 输出。来源可信（本空间已入库 chunk），但内容是**字符串注入**，内容里夹带的 `<` `>` `&` 会被当标签解析，且依赖「后端永远只输出 `<mark>`」这一隐含契约。
- 新版：标签外一律作为 React 文本子节点，**由 React 自动转义**，即使后端哪天多吐了别的标签也只会显示成字面文本，绝不会变成真实 DOM 节点。安全性从「信任约定」升级为「结构性免疫」。
